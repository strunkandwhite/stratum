# stratum

Shared Claude Code configuration across projects. Manages a global `CLAUDE.md`, accumulated permissions, and shared skills from a single repo.

## Setup

```bash
git clone https://github.com/strunkandwhite/stratum.git ~/code/stratum
cd ~/code/stratum
bash bin/sync.sh
```

The first run backs up existing `~/.claude/settings.json` and `~/.claude/CLAUDE.md` to `.bak` files, then symlinks them to this repo.

## What it does

`bin/sync.sh` (also available as `/stratum` in any Claude session):

1. **Pulls** the latest stratum changes from `origin/main`
2. **Gathers** permissions from all `~/code/*/.claude/settings.local.json` files
3. **Promotes** generic ones to the global `settings.json` (git, npm, pnpm, turso, MCP tools, etc.)
4. **Keeps** project-specific permissions local (unique API domains, env-prefixed commands, project paths)
5. **Trims** promoted entries from project files so they only contain what's truly local
6. **Verifies** symlinks from `~/.claude/` point here

WebFetch domains are promoted only if they appear in 2+ projects.

The `/stratum` command also commits and pushes any other outstanding changes in this repo after syncing.

## Structure

```
CLAUDE.md          # Global agent instructions (symlinked to ~/.claude/CLAUDE.md)
settings.json      # Global permissions (gitignored, generated by sync)
bin/sync.sh        # Sync script
commands/           # Shared skills (symlinked to ~/.claude/commands/)
  stratum.md       # /stratum slash command
```
